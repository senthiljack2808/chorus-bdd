# Chorus BDD

## Project Description

Chorus BDD is an open-source **Behavior-Driven Development (BDD)** testing framework designed to enable collaboration between developers, testers, and business stakeholders through readable, business-focused test scenarios.

It allows teams to define application behavior using **Gherkin-style feature files**, where requirements are expressed as human-readable scenarios. These scenarios are then executed as automated tests, ensuring that system behavior aligns with business expectations.

Chorus supports **end-to-end testing, integration testing, and acceptance testing**, and integrates well with Java-based automation frameworks. It provides built-in capabilities for **scenario execution, step definitions, reporting, and test lifecycle management**.

When integrated with **ReportPortal**, Chorus test executions can be visualized with enhanced reporting, logs, and analytics, improving debugging and test observability for engineering teams.

---

## Key Features

- Behavior-Driven Development using **Gherkin-style scenarios**
- Human-readable **feature files** for better collaboration
- Support for **Java-based step implementations**
- Built-in **reporting and test execution lifecycle**
- Integration with **CI/CD tools and test reporting platforms**
- Suitable for **integration, acceptance, and system testing**

---

## Use Cases

- Acceptance testing
- Integration testing
- End-to-end workflow validation
- Behavior-driven development in Java-based applications

## Integration

Chorus BDD can be integrated with modern test reporting platforms such as **ReportPortal** to provide:
